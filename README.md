# 操作系统作业调度算法实验
## 实验一：进程调度算法实现与分析

### 实验目标
本实验旨在通过编程实现三种经典的作业调度算法，并分析它们在不同环境下的性能表现：

1. **单处理器环境**：分析 n 个作业在单道环境下的调度性能，计算每个作业的开始时间、结束时间、周转时间、带权周转时间，以及整体平均性能指标
2. **多处理器环境**：在双道环境下重复上述分析，比较多道并行处理对调度性能的影响

### 实现算法
- **FCFS (First Come First Served)**：先来先服务调度算法
- **SJF (Shortest Job First)**：短作业优先调度算法  
- **HRRN (Highest Response Ratio Next)**：最高响应比优先调度算法

### 运行环境与编译

#### 代码运行
- **开发环境**：Rust 编程语言（推荐版本 1.70 及以上）
- **运行步骤**：进入 `task1` 目录执行以下命令

```bash
cargo run --quiet
```

程序将自动运行三种调度算法，输出单道和双道环境下的详细调度结果，包括各作业的时间统计和平均性能指标。

### 实验分析要求
1. 对比三种调度算法在相同作业批次下的平均周转时间和带权平均周转时间
2. 分析各算法的优缺点，讨论适用场景
3. 评估同一调度算法在不同作业流模式下的性能差异
